How this was researched — methodology & source conflicts

Every hard number below traces to a named, linked source. Where no single government or survey dataset breaks a metric out cleanly by these four specific job titles, that's disclosed explicitly rather than papered over with an invented figure — those cells carry an Estimate tag in the cards above.

Salary

Backend Software Engineer and Data Scientist salaries use the U.S. Bureau of Labor Statistics' Occupational Employment and Wage Statistics survey directly ("Software Developers" and "Data Scientists" occupational codes, May 2024 median annual wage). Neither "AI/ML Engineer" nor "Cloud Engineer" exists as a distinct BLS occupational code as of this data. For AI/ML Engineer, the figure is drawn from an industry analysis citing 2025 median compensation figures compiled from job-market data — flagged as an estimate. For Cloud Engineer, the closest official proxy is BLS "Network and Computer Systems Administrators" ($96,800, May 2024), but this is a conservative floor: it captures traditional on-prem sysadmin roles, not cloud-native titles, which several industry salary trackers (e.g. Robert Half) place noticeably higher. This is flagged as an estimate/proxy in the card.

Job growth & annual openings

Backend Software Engineer (+15%, ~129,200 openings/yr) and Data Scientist (+34%, ~23,400 openings/yr) again come straight from BLS Employment Projections, 2024–2034. AI/ML Engineer has no dedicated BLS series; the number shown is anchored to the closest related BLS occupation, "Computer and Information Research Scientists" (+26%, 2023–2033 STEM projections), supplemented with LinkedIn's 2026 Jobs on the Rise report, which ranked AI Engineer the #1 fastest-growing role in the U.S. by job-start volume. Cloud Engineer uses "Network and Computer Systems Administrators" as its BLS proxy, which is actually projected to decline 4% — a genuine data conflict, addressed below.

Conflict resolved — Cloud Engineer growth: The only BLS occupational code that maps to "cloud" work (network/systems administration) shows shrinking employment, because that code captures legacy on-prem IT admin roles being displaced by managed cloud services — not cloud engineering itself. Trade-press labor data (CompTIA State of the Tech Workforce 2025) describes this as a hollowing-out-and-shift, not a shrinking field: demand is migrating out of traditional sysadmin titles into cloud, DevOps, and security specializations. We kept the official BLS decline figure visible rather than silently swapping in a rosier number, and flagged the mismatch so it isn't mistaken for a verdict on cloud engineering demand overall.

Job postings volume

Indeed Hiring Lab's 2026 labor-market data shows overall U.S. software development postings up almost 15% since February 2025, alongside a broader ~7% decline in postings overall — with the rebound concentrated in senior and AI-titled roles. Indeed does not publish a clean postings count broken out identically across all four of these specific titles, so the relative postings-volume scores in this tool are a qualitative estimate informed by that report, BLS "number of jobs" base figures, and LinkedIn's 2026 rankings — flagged accordingly.

Learning difficulty / time to job-ready

This is inherently qualitative and is flagged as an estimate throughout. It's grounded in BLS's stated typical entry-level education per role (bachelor's degree across all four, with BLS explicitly noting employers "require or prefer" a master's or doctoral degree for data scientists) plus the widely reported fact that AI/ML engineering requires backend engineering skills as a prerequisite layer on top of applied math, statistics, and deep learning frameworks — making it the most cumulative path of the four.

Remote-work availability

The Stack Overflow 2025 Developer Survey reports 45% of U.S. developers work remotely — the only hard, citable, cross-role U.S. figure available. It does not break this out separately by AI/ML, backend, data science, or cloud titles, so the relative differences shown between the four paths are a qualitative estimate based on general industry observation (cloud/DevOps and data roles skew slightly more distributed than early-career backend roles) and are flagged as such.

AI-automation / career-longevity risk

Indeed Hiring Lab's GenAI Skill Transformation Index (2025) and a payroll-data study (cited via SoftwareSeni, using the Eloundou et al. AI-exposure framework) both place software engineering skills in a high-exposure quintile for GenAI-driven transformation — yet Indeed's own 2026 data shows software postings rebounding faster than the market since agentic coding tools launched, complicating a simple "high risk = bad" read. No equivalent named study scores AI/ML Engineer, Data Scientist, and Cloud Engineer against the same index, so those three are qualitative, source-informed estimates and are flagged.

What's solid vs. what's an estimate, at a glance

  • Fully sourced, exact: Backend Software Engineer and Data Scientist — salary, growth rate, annual openings (all BLS).
  • Proxy-sourced (real number, imperfect title match): AI/ML Engineer and Cloud Engineer — salary and growth, using the nearest official or named industry figure.
  • Qualitative estimate, source-informed: Learning difficulty, remote-work availability, and automation risk for all four roles; postings volume for AI/ML and Cloud.
